Getting Started - Quick Intro

If returning to ain't, choose OPEN PROJECT and select from the AINT project files.

If new to ain't...

  1. Choose NEW PROJECT.
  2. In the 'Save As' prompt, enter a name for your project and save.
  3. Enter a name for your first entry and choose ADD.
  4. Write a great novel.

AUTOSAVE will save your work every minute, or CTRL+S to save as you please. You may also SAVE AS another project file.

ADD as many entries as you'd like; they will all be listed to the right. You can navigate by directly selecting the entry.

RENAME as often as you'd like.

DELETE the entry if you hate it. Or, before you DELETE, create an 'unwanted' entry to hold all unwanted material.

IMPORT (open) text or Html files to create new entries.

EXPORT (save) the entry to text or Html if you desire to use outside of ain't. If you choose to save as a text file (.txt) ain't will attempt to retain a readable text version of styled elements (lists, tables, rules and headers). Exporting Html is experimental and should only be used if you want a styled copy of the entry to use/reference outside of ain't. Formatting should be maintained when copying and pasting into Google Docs™.

EXPORT PROJECT will save all entries in a single text or HTML file.

The controls are fairly self-explanatory (refer to the table at the bottom of this page for a full list with descriptions and hotkeys).

Also refer to the list at the bottom of this page to view a list of changes, fixes, and additions from version 1.

I created this app to accommodate my way of writing. Certain elements have been intentionally omitted. G-Drive™ synchronization is not something I've embraced with Chromebooks™; I like to have control of my G-Drive™. Loading of files should also be within the user's control. This is why you are required to open a project on start-up, or save a new project before creating an entry. Autosave only works on files you have explicitly defined; ain't never creates files on its own. I have added a few special elements that are not often found in simple text editors (at least not those on Chromebooks™). Besides full-screen (no distraction) editing, you may also choose to minify the editor and place it on top of all other windows. This is useful when working with other apps or while referencing and researching. You may also split the screen between the current entry and a universal notepad. ain't also allows you to set the font size and color, and background color of the editor. Saving is automatic with almost every action, even with autosave OFF. You should only need to open ain't, write, and close it when finished. If you are like me, you never 'shut down' and could leave ain't open all the time; although, I do suggest closing ain't on occasion to allow Chromium™ to refresh. As far as bugs go, they are to be expected... until Google™ buys me out. :)

NOTE: ain't is a rich text editor, in that it can format text with "rich text" elements. ain't cannot open, nor save, RTF (Rich Text Files).

Easily create tables

Select the table icon () in the tools menu to add an empty table to the entry; this will initially appear as a square. Choose the square to open the table editor. Comma separated values define the table cells. The first line will be your table header (fruit,color,taste). Enter clears the table header and creates the next row (apple,red,good). Enter again and again to create a full table. Keep all rows on one line; enter will always create a new row.

This...

fruit,color,taste
apple,red,good
banana,yellow,gross
blueberry,blue,I'm allergic... will kill me

will create this...

fruit color taste
apple red good
banana yellow gross
blueberry blue I'm allergic... will kill me

Tables can only be styled within the CSV editor.

This...

fruit,color,taste
apple,<u>red</u>,good
banana,<b>yellow</b>,gross
blueberry,<i>blue</i>,I'm allergic<hr>will kill me

will create this...

fruit color taste
apple red good
banana yellow gross
blueberry blue I'm allergic
will kill me
